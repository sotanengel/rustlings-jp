# Error handling

ほとんどのエラーはプログラムを完全に止めるほど危険なものではない。
関数が失敗した時、その失敗は容易に説明し対応できる理由によるものであることがある。
例えば、ファイルがないためにファイルを開く処理が失敗した場合には処理を終了する代わりにファイルを作るようにしたいかもしれない。
